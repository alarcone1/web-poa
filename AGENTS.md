# OpenCode Rules for Escuela ETDH Project

## 1. Project Context
Este proyecto utiliza la metodología **Desarrollo Guiado por Contexto (DGC)**.
- Los archivos de contexto (ADN del proyecto) están en `docs/context/`.
- Siempre debes leer `docs/context/PRD.md`, `docs/context/ESTILO.md`, `docs/context/TARGET.md` y `docs/context/HISTORIAS.md` antes de responder.

## 2. Workflow
- Trabajamos **una Historia de Usuario (HU) a la vez**.
- La HU activa tiene el estado `[/]` en `docs/context/HISTORIAS.md`.
- Antes de codificar, debes presentar un plan de acción (Protocolo de Destilación).

## 3. Communication Style
- Sé directo, informativo y sin rodeos.
- No uses exclamaciones, sarcasmo ni lenguaje de marketing.
- Siempre confirma antes de escribir código.

## 4. Technology Stack
- HTML5, CSS3, JavaScript (Vanilla)
- Tailwind CSS v4 (vía CDN)
- Sin backend ni base de datos
